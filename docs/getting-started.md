# Getting Started with ContriBot

Welcome to ContriBot! This guide will help you get started with using ContriBot to manage your GitHub repositories.

## Prerequisites
- A GitHub account.
- Access to a repository that has ContriBot enabled.

## Step 1: Sign In
1. Open the ContriBot dashboard URL.
2. Click **Sign In** and authenticate using your GitHub account.
3. You will be redirected to the ContriBot Dashboard.

## Step 2: Adding a Repository
1. On the Dashboard, click **Add Repository**.
2. Enter the full name of the repository you want ContriBot to manage (e.g., `your-username/your-repo`).
3. Ensure that you have the necessary permissions on the repository.

## Step 3: Enabling ContriBot
1. Once the repository is added, click the **Settings** gear icon next to it.
2. Toggle **Enable Webhooks** to active. ContriBot will automatically configure the necessary webhooks in your GitHub repository.

## Step 4: Using ContriBot
1. Create a new issue in your repository.
2. ContriBot will automatically analyze the issue, classify it, and wait for your approval to begin implementation.
3. Once approved, ContriBot will write the code, open a Pull Request, and perform multi-model verification.
4. Review the Pull Request and merge it when you are ready.
